Detailed Description

The XWindow class encapsulates an X lib Window object, which represents the actual window users see and interact with.

It is derived from Component, and thus shares much of its functionality, though it must override some of the functionality to make it appropriate to a Window (such as setting the location and bounds).

This class also maintains a mapping of X lib window pointers to our XWindow class pointers. Thus, when X lib creates a new event and specifies a window, you can retrieve the XWindow object corresponding to that window.

Each XWindow object maintains its own Graphics context, which it passes on to its child components to enable drawing into the window.

PaintEvents are passed to the XWindow. Paint events should be analyzed and passed on to children to have them paint themselves. However, only children intersecting the damaged area specified by the PaintEvent should be drawn.
